# Kano 二維品質模式互動遊戲

這是一個用來教 Kano 二維品質模式的四關卡網頁互動遊戲。

## 學習目標

學生完成遊戲後，能理解：

1. 基本品質、一元品質、魅力品質、無差異品質、反轉品質的差異。
2. 服務品質與媒體／活動設計中，不是所有功能都越多越好。
3. 桃園機場案例可用來理解服務品質如何被分類。
4. 校園活動案例可用來理解 Kano 模型如何轉化為體驗設計。
5. Kano 模型不只是分類工具，也是品質策略與資源配置工具。

## 四個關卡

1. Kano 分類快問快答
2. 桃園機場服務品質
3. 校園 AI 素養體驗日
4. 預算有限的品質策略

## 部署方式

到 GitHub repo 的 Settings → Pages，選擇：

- Source: Deploy from a branch
- Branch: main
- Folder: /root

儲存後即可部署為 GitHub Pages。

## 教學建議

- 5 分鐘：教師說明 Kano 五種品質屬性。
- 15 分鐘：學生兩人一組完成遊戲。
- 10 分鐘：討論桃園機場案例中，哪些服務是基本品質，哪些是魅力品質。
- 15 分鐘：讓學生把自己的活動或媒體專案套用 Kano 模型。
- 10 分鐘：收斂品質策略：先保底，再提升，最後創造驚喜。
